執筆ルール
このドキュメントは、docs配下の記事を分析して抽出した文体・構成・表現の特徴をまとめたものです。新しい記事を執筆する際は、このルールに従って執筆してください。

1. 文体の基本方針
1.1 敬体の使用
- 「です・ます」調を徹底する
- 丁寧で柔らかい表現を心がける
- 読者に語りかけるような文体を維持する
- 読者に問いかける文の末尾には「？」をつける
例：
- ○「この記事では、そんなRAGの詳細について解説していきます。」
- ×「この記事では、RAGの詳細を解説する。」
- ○「という経験はありませんか？」
- ×「という経験はありませんか。」

1.2 論理的で段階的な説明
- 課題提示 → 解決策 → 効果という流れを基本とする
- 接続語を適切に使用する
  - 「これにより」「したがって」「また」「さらに」「一方」「しかし」
例：
- 「RAGは、大規模なデータセットから関連情報を取得し（Retriever）、その後に生成モデル（Generator）がその情報を元にテキストを生成します。これにより、従来より正確で精度の高い情報提供が可能になります。」

1.3 バランスの取れた視点
- メリットだけでなく、課題や限界も必ず提示する
- 「しかし」「一方」などで対比を明確にする
- 客観的で公平な視点を保つ

1.4 体験・考察の挿入
- 適度に執筆者の体験や考察を混ぜることで、読者が内容をイメージしやすくする
- 全てのセクションに入れる必要はない（冗長になるため）
- 読者の共感を得やすい場所や、抽象的な内容を具体化したい場所に配置する
- 「私自身、～という経験があります」「実際に～してみると」などの表現を活用する
例：
- 「私自身、初めてChatGPTを使い始めた頃は、なぜ時々的外れな回答が返ってくるのか理解できませんでした。しかし、プロンプトの書き方を学んでからは、AIとのやり取りの質が劇的に向上しました。」

2. 記事構成のテンプレート
2.1 導入部分
1. タイトルとサブタイトル
   - メインタイトル：簡潔で分かりやすく
   - サブタイトル：記事の内容を補足する説明
2. 概要説明（3-4段落）
   - 第1段落：テーマの定義と重要性
   - 第2段落：背景や課題の説明
   - 第3段落：記事の目的や内容の予告
3. メタ情報
   - カテゴリ
   - 著者名
   - 読了時間
4. 目次
   - 主要セクションのリスト

2.2 本文の構成
- 大見出し（H2）で章を分ける
  - 各章は明確なテーマを持つ
  - 論理的な順序で配置する
- セクションのリード文
  - 各主要セクション（H2）の冒頭に、そのセクションを読む意義や目的を説明するリード文を配置する
  - 「なぜこの話をするのか」を明確にすることで、読者のモチベーションを維持する
  - リード文は1-2段落程度で簡潔に
  - 全てのセクションに必須ではないが、重要なセクションには配置を推奨
- 小見出し（H3）で細分化
  - 各大見出しの下に複数の小見出し
  - 具体的で分かりやすい見出しにする
- セクション末尾のナビゲーション
  - 各主要セクションの最後に「TOPへ」のリンク

2.3 まとめ
- 記事の要点を再確認
- 今後の展望や期待される発展を示す
- 読者に次のアクションを促す（該当する場合）

3. 段落と文章の書き方
3.1 段落構成
- 1段落は2-4文程度を目安とする
- 長い説明の後は必ず空行を入れる
- 改行を適切に使い、読みやすさを重視する

3.2 文の長さ
- 1文は比較的短く保つ
- 複雑な概念は複数の文に分けて段階的に説明する
- 読点（、）を適切に使用し、リズムを作る

3.3 段落のパターン
定義型：
- 第1文：定義や概要
- 第2文：背景や理由
- 第3文：効果や意義
例示型：
- 第1文：一般的な説明
- 第2文：「例えば」で具体例を提示
- 第3文：具体例からの効果や学び

4. 専門用語の扱い方
4.1 初出時の説明
- 専門用語は必ず初出時に説明を加える
- カタカナ語や英語の略語には原語と読み方を併記する
- 括弧を使って補足説明する
例：
- 「RAG（Retriever-Augmented Generation）は、「ラグ」と呼ばれる...」
- 「GPT（Generative Pre-trained Transformer）は...」

4.2 用語の一貫性
- 同じ概念には同じ用語を使用する
- 略語と正式名称を適切に使い分ける
- 初出以降は読者が理解しやすい形式を選択する

4.3 関連用語へのリンク
- 用語集が存在する場合はリンクを設置する
- 内部リンクを活用して読者の理解を深める
例：
- 「▼ChatGPTとは？」
- 「⇒ ChatGPT｜用語集」

5. 具体例の提示方法
5.1 「例えば」の活用
- 抽象的な概念の後には必ず具体例を提示する
- 「例えば、」で始める
- 実際のビジネスシーンや現実の状況を想定する
例：
- 「例えば、ビジネスにおいて生成AIで生産性を高めたいと考えた時、自社に蓄積された社内情報や外部の最新情報をも取り込むことができれば...」

5.2 具体例のパターン
抽象的説明 + 例えば、[具体的なシナリオ]. [結果や効果].

5.3 複数の具体例
- 医療、金融、カスタマーサポートなど、異なる分野での応用例を示す
- 読者が自分の状況に当てはめやすくする

6. 比較・対比の表現方法
6.1 表形式での比較
- 2つ以上の技術や手法を比較する際は表を使用する
- 比較項目を明確に設定する
- 各項目について簡潔に説明する
比較項目の例：
- 知識の更新
- 情報の信頼性
- 応答の内容
- 利用シーン
- 計算リソース
- メリット・デメリット

6.2 対比の接続語
- 「一方、」：対照的な内容を示す
- 「対して」：比較対象との違いを強調
- 「しかし」：前述内容との転換
- 「それに対し」：明確な対比
例：
- 「GPTは...。一方、RAGは...」

7. 視覚要素の活用
7.1 画像の挿入
- 各主要セクション（H2）に1つの画像を配置
- 画像ファイル名は内容を反映した分かりやすいものにする
- 画像は説明の補助として機能させる
命名例：
- img_column_data-utilization-genai-rag_01.png
- img_column_data-utilization-genai-rag_02.png

7.2 表の使用
- 比較情報は表形式で整理する
- 複雑な情報を視覚的に理解しやすくする
- 表の各セルは簡潔に記述する

7.3 箇条書き
- 複数の項目を列挙する際は箇条書きを使用
- 「-」（ハイフン）で項目を示す
- 各項目は簡潔にまとめる

8. 頻出表現パターン
8.1 導入・予告の表現
- 「この記事では、～について解説していきます。」
- 「では、具体的に～を見ていきましょう。」
- 「本セクションでは、～について詳しく解説します。」
- 「この章では、～を考察していきます。」

8.2 説明の展開
- 「～により、」「これにより、」
- 「したがって、」「そのため、」
- 「また、」「さらに、」
- 「特に、」「具体的には、」

8.3 効果・可能性の表現
- 「～が可能になります」
- 「～が期待されています」
- 「～することができます」
- 「～を実現できます」
- 「～に役立ちます」
- 「～が図れます」

8.4 課題・注意点の表現
- 「～に注意する必要があります」
- 「～が求められます」
- 「～が重要です」
- 「～が欠かせません」

8.5 例示の表現
- 「例えば、」
- 「具体的には、」
- 「例として、」

9. 避けるべき表現
9.1 カジュアルすぎる表現
- ×「～だね」「～だよ」
- ×「～しちゃう」「～ちゃった」
- ×「めっちゃ」「超」
- ○フォーマルな文体を維持する

9.2 断定的すぎる表現
- ×「必ず～します」
- ×「絶対に～です」
- ×「間違いなく～」
- ○「～が可能です」「～が期待されています」など、やや柔らかい表現を使用

9.3 曖昧な表現
- ×「いろいろな」「さまざまな」の多用
- ×「など」だけで終わる文
- ○具体的な例を挙げる

9.4 専門用語の無説明使用
- ×初出時に説明なしで専門用語を使用
- ○必ず初出時に定義や説明を加える

9.5 長すぎる文章
- ×1文に複数の主張を詰め込む
- ○1文1メッセージを基本とする
- ○複雑な内容は複数の文に分ける

10. 技術説明の標準フロー
技術やコンセプトを説明する際は、以下の順序で展開する：
1. 定義：その技術が何であるかを明確に定義する
2. 歴史/背景：なぜその技術が生まれたのか、背景を説明する
3. 仕組み：どのように機能するのかを説明する
4. メリット：どのような利点があるのかを列挙する
5. 課題：どのような課題や限界があるのかを提示する
6. 応用例：実際にどのような場面で使われるのかを具体的に示す
7. 今後の展望：将来どのように発展していくかを考察する

11. 読者への配慮
11.1 専門知識レベルの想定
- 技術に興味があるが専門家ではない読者を想定
- 基本的な概念から丁寧に説明する
- 専門家にとっても有益な深い情報も含める

11.2 実用性の重視
- 理論だけでなく、実際のビジネスや業務での活用方法を示す
- 読者が自分の状況に当てはめて考えられるようにする

11.3 ナビゲーションの配慮
- 長い記事では適宜「TOPへ」リンクを配置
- 目次を設置して全体像を把握しやすくする
- 関連する用語集や他の記事へのリンクを提供する

12. 記事タイトルの付け方
12.1 メインタイトル
- 簡潔で分かりやすく（10-20文字程度）
- 疑問形を使うことが多い
- 「～とは？」パターンが頻出
例：
- 「RAG（検索拡張生成）とは？」

12.2 サブタイトル
- メインタイトルを補足する説明
- 記事の内容や価値を明確にする
例：
- 「生成AIの新しいアプローチを解説」

17. リサーチについて
もし執筆時にリサーチが必要、あるいはユーザから明示的にリサーチを求められた場合、あなたは下記のドメインにのみアクセスを許可されています。これらのドメインに絞って検索を行ってください。

18. 記事執筆に取り掛かる前に：一度構成を書いてからレビューをもらう
記事執筆の依頼をユーザから受け取った際に、いきなり執筆に取り掛かるのではなく、まずはユーザに構成案を提案して、レビューをもらいましょう。ユーザが後からするフィードバックを減らすために早い段階でフィードバックを受けることが望ましいです。
プロセス：
1. リサーチの実施
   - 必要に応じて情報収集を行う
   - 既存の関連記事を確認する
2. 構成案の作成
   - 記事のタイトル（メインとサブタイトル）を提案
   - 主要セクション（H2）の見出しを列挙
   - 各セクションで扱う内容を簡潔に説明（1-2文程度）
   - 想定読了時間を提示
3. ユーザへの提示
   - 作成した構成案をユーザに提示する
   - レビューとフィードバックを依頼する
4. フィードバックの反映
   - ユーザからの指摘を構成案に反映する
   - 必要に応じて再度レビューを依頼する
5. 執筆開始
   - 承認された構成案に基づいて本文を執筆する

構成案の例：
【記事構成案】

タイトル：「○○○とは？」
サブタイトル：「△△△について解説」
読了時間：約10分

## 目次案

1. ○○○とは何か
   - 定義と基本概念
   - なぜ注目されているのか

2. ○○○の仕組み
   - 基本的なアーキテクチャ
   - 動作の流れ

3. ○○○のメリットと課題
   - 主な利点（3-4点）
   - 注意すべき課題（2-3点）

4. 実践的な活用例
   - ビジネスでの活用
   - 具体的なユースケース

5. まとめ
   - 要点の再確認
   - 今後の展望

この構成でよろしいでしょうか？ご意見やご要望があればお聞かせください。

メリット：
- 執筆方向性の早期確認により手戻りを防ぐ
- ユーザの期待と執筆内容のズレを最小化
- 効率的な執筆プロセスの実現

執筆時のチェックリスト
記事執筆後、以下の項目を確認してください：
- 「です・ます」調が一貫しているか
- 専門用語は初出時に説明されているか
- 具体例が適切に配置されているか
- メリットだけでなく課題も提示されているか
- 段落の長さは適切か（2-4文程度）
- 接続語が適切に使用されているか
- 表や画像が効果的に配置されているか
- 目次が設置されているか
- 各主要セクションの最後に「TOPへ」リンクがあるか
- まとめのセクションで要点を再確認しているか
- 読了時間、著者情報などのメタ情報が含まれているか
- 関連用語へのリンクが適切に設置されているか

このルールに従って執筆することで、読者にとって理解しやすく、専門的でありながら親しみやすい技術記事を作成できます。

フィードバック反映の作業ルール
記事の構成や内容にフィードバックをもらった場合、以下のプロセスで対応してください：
1. フィードバックの理解
   - 具体的なフィードバック内容を正確に把握する
   - 該当箇所を特定し、改善点を明確にする
2. 記事への反映
   - フィードバックに基づいて記事を修正する
   - 修正箇所が複数ある場合は、漏れがないよう確認する
3. 抽象化と学習
   - 個別のフィードバックから一般化できる原則や学びを抽出する
   - 「このフィードバックは他の記事にも適用できるか？」を考える
4. 執筆ルールへの反映
   - 抽象化した学びをこのCLAUDE.mdに追記する
   - 既存のルールと矛盾がないか確認する
   - 適切なセクションに配置する（新規セクション作成も可）
5. 継続的改善
   - フィードバックを受けるたびにこのプロセスを繰り返す
   - 執筆ルール自体も進化させ続ける

このプロセスの目的：
- 個別のフィードバックを組織知として蓄積する
- 同じ指摘を繰り返し受けることを防ぐ
- 記事の品質を継続的に向上させる

# 執筆ガイドライン

- 各主要セクションの冒頭には、その話題がなぜ重要なのかを伝えるリード文を一文添えてください。
- 読み手のイメージが膨らむと判断した場合は、簡潔な自身の経験や考察を織り交ぜてください。ただし、すべてのセクションに無理に入れる必要はありません。
- 疑問文の末尾は必ず全角の疑問符（？）で終わらせてください。
- 文章全体のトーンは、情報性を保ちつつ親しみやすい語り口とストーリーテリングの要素を両立させてください。
